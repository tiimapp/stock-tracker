## 📋 系统概述

C2605 玉米期货监控系统是一个自动化期货价格监控和报告系统，为 Discord 用户提供实时价格快报和每日总结。

### 核心功能

1. **实时价格监控** - 交易时段内每小时更新
2. **每日总结报告** - 收盘后生成 OHLCV 总结
3. **夜盘状态验证** - 每日自动验证夜盘交易状态
4. **交易日识别** - 自动识别工作日和节假日
5. **多源数据降级** - 多层数据源确保可靠性

## 🔄 核心流程

### 1. 盘中快报流程 (Hourly Report)

```
触发：Cron 定时任务 (09:00, 10:00, 11:00, 14:00, 15:00)
       │
       ▼
┌─────────────────────────────┐
│ 1. 加载配置                  │
│    load_config()            │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│ 2. 检查是否交易日            │
│    is_trading_day()         │
└──────────────┬──────────────┘
               │ 非交易日 → 退出
               │ 交易日 ↓
               ▼
┌─────────────────────────────┐
│ 3. 检查是否在交易时段        │
│    is_trading_time()        │
└──────────────┬──────────────┘
               │ 非交易时段 → 退出
               │ 交易时段 ↓
               ▼
┌─────────────────────────────┐
│ 4. 获取价格数据              │
│    fetch_c2605_data()       │
│    P0 → P1 → P4 降级        │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│ 5. 格式化报告                │
│    format_report(type=      │
│      'hourly')              │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│ 6. 发送到 Discord            │
│    send_to_discord()        │
└──────────────┬──────────────┘
               │
               ▼

│ 7. 写入日志文件              │
```

```
               ▼
```

## 📅 调度系统

### Cron 定时任务

| 任务类型 | 时间 | Cron 表达式 | 说明 |
|---------|------|------------|------|
| 盘中快报 | 09:00 | `0 9 * * 1-5` | 早盘开始 |
| 盘中快报 | 10:00 | `0 10 * * 1-5` | 早盘中段 |
| 盘中快报 | 11:00 | `0 11 * * 1-5` | 午盘前 |
| 盘中快报 | 14:00 | `0 14 * * 1-5` | 下午盘 |
| 盘中快报 | 15:00 | `0 15 * * 1-5` | 收盘时 |
| 每日总结 | 15:30 | `30 15 * * 1-5` | 收盘后总结 |
| 夜盘快报 | 21:00 | `0 21 * * 1-5` | 夜盘 (待确认) |
| 夜盘快报 | 22:00 | `0 22 * * 1-5` | 夜盘中 (待确认) |
| 夜盘快报 | 23:00 | `0 23 * * 1-5` | 夜盘结束 (待确认) |

**注意:** 夜盘报告当前状态为 **UNCERTAIN**，继续监控中。

### 心跳检查

| 检查项 | 频率 | 时间 | 脚本 |
|--------|------|------|------|
| 交易日验证 | 每日 1 次 | 首次心跳 (<10:00) | `heartbeat_trading_check.py` |
| 夜盘验证 | 每日 1 次 | 首次心跳 | `dce_trading_verifier.py` |
| 系统状态 | 每 2 小时 | 心跳触发 | OpenClaw 内置 |
| 模型健康 | 每 2 小时 | 心跳触发 | OpenClaw 内置 |

## 🛡️ 容错与降级

### 数据源降级链

```
新浪实时 API (P0)
    ↓ 失败 (超时/解析错误)
AKShare 历史数据 (P1)
    ↓ 失败 (库不可用/网络错误)
Mock 数据 (P4)
```

### 验证降级策略

- **Tavily 不可用** → 使用 Perplexity 搜索结果
- **所有搜索失败** → 保持上次验证状态 (night_session_enabled: null)
- **状态文件损坏** → 使用默认值 (继续监控模式)

### 报告降级策略

- **Discord 发送失败** → 记录日志，不重试
- **数据过期** → 在报告中标注"数据延迟"警告
- **非交易日** → 静默跳过，不发送报告
